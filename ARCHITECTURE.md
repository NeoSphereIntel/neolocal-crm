## 1. Stack & Deployment

| Layer | Technology |
|---|---|
| Runtime | Google Apps Script (V8) |
| Database | Google Sheets ("Leads Master" — ~100 columns) |
| Frontend (current) | Apps Script HtmlService templates |
| Frontend (target) | Decoupled static site calling Apps Script JSON API |
| External APIs | SerpAPI (Google Maps import), no LLM calls |
| Deployment | `clasp push` → Apps Script editor → deploy as web app |
| Secrets | Script Properties only (`PropertiesService.getScriptProperties()`) |
| Timezone | America/Toronto (appsscript.json) |

No `require`, no `import`, no `fetch`, no `async/await`. All functions are global scope.

## 2. Sheets (Database Layer)

| Sheet name constant | Default name | Purpose |
|---|---|---|
| `APP.SHEETS.LEADS` | Leads Master | Primary CRM data — ~100 columns, one row per lead |
| `APP.SHEETS.SEARCH_CONFIG` | Search Config | SerpAPI search jobs (query, city, country, active flag) |
| `APP.SHEETS.IMPORT_LOG` | Import Log | Record of each import run |
| `APP.SHEETS.SHADOW_JOBS` | Shadow Jobs | Generated shadow job briefs |

**Column access pattern:** always by header name via `getHeaders_()` / `getRowObject_()` — never by column index.

**Key column families in Leads Master:**
- Identity: `lead_id`, `lead_signature`, `business_name`, `category`, `city`, `province_state`, `country`
- SerpAPI signals: `reviews_count`, `rating`, `place_id`, `maps_url`
- Competitor data: `comp_1_name` … `comp_5_name`, `comp_avg_reviews`, `comp_max_reviews`
- Snapshot scores: `score_base_presence`, `score_trust`, `score_competitive_pressure`, `score_opportunity`, `score_difficulty`, `score_priority`
- Snapshot intelligence: `diagnosis_state`, `priority_bucket`, `momentum_state`, `is_undervalued`, `snapshot_narrative`, `market_position_summary`, `strategic_gap_summary`, `action_implication_summary`
- CRM workflow: `lead_status`, `assigned_rep`, `last_updated_at`, `last_activity_log`
- Task: `active_task`, `task_type`, `task_due_at`, `task_status`
- Outreach: `outreach_message`, `reply_message`
- Shadow jobs: `shadow_job_status`, `shadow_job_title`, `shadow_job_created_at`
- Operator layer: `operator_reviews_goal`, `operator_current_gmb_status`, `operator_claimed`, `operator_mismatch_flags`

## 3. File-by-File Map

### `00_config.js`
Global `APP` constant. Defines sheet names, `MAX_RESULTS_PER_SEARCH` (20), `TOP_COMPETITOR_COUNT` (5), `SNAPSHOT_VERSION`, `LEAD_STATUS_DEFAULT`, `MARKET_MIRROR_WEBAPP_URL`, country/province-state lists. Referenced by every other file.

### `01_menu_and_setup.js`
- `onOpen()` — builds NeoLocal spreadsheet menu
- `buildNeoLocalSalesEngineV23()` — master setup runner
- Sheet setup: `setupSearchConfigSheet_`, `setupLeadsMasterSheet_`, `setupImportLogSheet_`, `setupShadowJobsSheet_`
- Formatting: `formatAllSheets`, `applyBaseSheetStyle_`, `styleLeadsSheet_`, etc.
- Snapshot rebuild menu wrappers: `rebuildAllSnapshots`, `rebuildSelectedLeadSnapshot`

### `02_import_engine.js`
SerpAPI bulk import pipeline.
- Entry point: `runActiveSearchImports`
- Chain: `readSearchConfigs_` → `importOneSearchConfig_` → `fetchSerpApiLocalResults_` → `normalizeLocalResults_` → per-result: `buildNormalizedLeadObject_` + `buildMarketContext_` + `calculateCompetitorSignals_` + `generateSnapshot_` (from 03) → `appendNewLead_` or `updateExistingLeadFromImport_`
- After all imports: calls `refreshCRMExecutionLayer` (from 10)
- Also calls `styleLeadsSheet_` (from 01) after sheet writes

### `03_intelligence_engine.js`
Core scoring and classification engine.
- `generateSnapshot_(lead)` — main orchestrator: builds metrics → scores → diagnosis → narrative package
- `buildSnapshotMetrics_` — computes `momentum_state` via `computeMomentumSignal_`, `is_undervalued` via `computeUndervalued_`
- `calculateLeadScores_` — produces 6 scores: base_presence, trust, competitive_pressure, opportunity, difficulty, priority
- `determineDiagnosisState_` — maps scores to: Invisible, Outgunned, Emerging, Undersignaled, Contender, Anchor
- `buildSnapshotNarrativePackage_` — delegates to 04
- Auto-retail branch: `determineAutoRetailDiagnosisState_`, `classifyAutoRetailPriority_`
- Operator layer: `buildOperatorExpectationLayer_`, `detectOperatorMismatch_`
- Backward-compat wrappers: `buildSnapshotForLeadRow_`, `buildSnapshotFromMetrics_`

### `04_snapshot_narrative.js`
Rule-based text narrative generation (no LLM).
- Entry: `buildSnapshotNarrativePackage_(metrics, scores, vertical)` — called from 03
- Routes on vertical: auto_retail → `buildAutoRetailSnapshotNarrativePackage_`; others → generic builders
- Returns 4 fields: `market_position_summary`, `strategic_gap_summary`, `action_implication_summary`, `snapshot_narrative`
- Operator mismatch text: `buildOperatorMismatchSummary_` calls `detectOperatorMismatch_` from 03

### `05_shadow_jobs.js`
Shadow job brief generator.
- Menu entry points: `generateShadowJobForSelectedLead`, `generateShadowJobsForLeadsWithoutOne`
- Core: `createShadowJobObject_` builds market hook + gap angle + action angle + draft prompt
- Writes to Shadow Jobs sheet via `appendShadowJob_`; back-fills Leads Master via `writeRowUpdates_`
- Consumes `momentum_state` and `is_undervalued` signals from lead row

### `06_sheet_helpers_and_utils.js`
Foundation utility layer — used by all other files.
- Sheet access: `getSheetDataObjects_`, `getHeaders_`, `getRowObject_`, `writeRowUpdates_`, `setHeadersIfNeeded_`
- ID generation: `generateLeadId_` (MD5-based), `makeLeadSignature_` (slugified name|city|category)
- URL builders: `buildMarketMirrorUrl_`, `buildRepSupportUrl_`
- Admin/repair: `ensureLeadsColumn_`, `hardResetSearchConfigSheet_`, `hardResetLeadsFormats_`, `enforceLeadsNumericFormats_`
- Utilities: `getScriptPropertyOrThrow_`, `md5Hex_`, `toQueryString_`, `classifyMarketPressure_`, `nowStr_`, `safeText_`, `truncate_`

### `07_vertical_profiles.js`
Vertical routing and profile definitions.
- `determineVerticalType_(obj)` — keyword-scans `category`/`niche`/`query` → returns one of 8 vertical keys
- `getVerticalProfile_(verticalKey)` — returns profile object with: `label`, `trust_driver`, `primary_risk`, `default_choice_language`, `proof_language`, `offer_language`, `template_family`
- Template families: `auto_retail`, `local_service`, `home_project`
- Called by 03, 04, 08, 09

### `08_outreach_engine.js`
Outreach message generator.
- `generateOutreachMessage_(m, diagnosis, scores)` — routes to `buildAutoRetailOutreach_` or `buildGenericOutreach_`
- Body assembled from: `getMarketContextLine_`, `getOutreachAngleFromDiagnosis_`, `getMomentumOutreachLine_`, `getUndervaluedOutreachLine_`
- `rebuildAllOutreachMessages()` — bulk rebuild menu function
- `getSignatureName_()` — derives name from active user's email

### `09_reply_engine.js`
Inbound reply handler.
- `generateReplyMessage_` — classifies inbound text via `classifyInboundReply_` (6 types: curious, recognized, skeptical, challenge, offer_question, neutral)
- Routes to `buildAutoRetailReply_` or `buildGenericReply_`
- `rebuildAllReplyMessages()` — bulk rebuild menu function

### `10_crm_workflow.js`
Full CRM state machine and action engine.
- `refreshCRMExecutionLayer` — master refresh: ensure columns → compute state → rebuild workspace/follow-up/pipeline views
- Public action functions (by lead_id): `markAsContacted`, `markAsReplied`, `markAsQualified`, `markCallBooked`, `markSnapshotSent`
- Menu wrappers: `markSelectedLeadAs*`
- Core: `applyCRMActionByLeadId_` — find row → guardrails → update → log activity → refresh views → toast
- Views: `buildSalesWorkspace_`, `buildFollowUps_`, `buildPipeline_`
- Follow-up cadence: 1st=2d, 2nd=3d, 3rd=5d, 4th+=7d

### `11_crm_sidebar.js`
Spreadsheet sidebar — bridge between sheet UI and backend.
- `openCRMActionSidebar()` — serves `crm_action_sidebar.html`
- Backend functions: `getSelectedLeadContextForSidebar`, `runCRMActionFromSidebar` (proxies to 10), `getMarketMirrorHtmlForSidebar` (uses 16+17+18)
- `installCRMPanelMenuTrigger()` — sets up installable trigger (run once from editor)

### `12_market_mirror_schema.js`
Market Mirror input schema and normalization.
- `MM.DEFAULTS` — full input schema with defaults
- `normalizeMarketMirrorInput_` — validates and normalizes raw input
- `determineMarketMirrorVertical_` — vertical detection for Market Mirror subsystem
- Called by 16

### `13_market_mirror_scores.js`
Market Mirror scoring tables and computation.
- `MM.SCORING` — scoring band tables
- `calculateMarketMirrorDerived_` — computes 7 core metrics + bands
- Score functions: `scoreReviewStrength_`, `scoreRatingStrength_`, `scorePresenceCompleteness_`, `scoreProofOfWork_`, `scoreMarketWeight_`, `scoreBuyerConfidence_`, `scoreVisibilityPressure_`, `scoreAccelerationPotential_`
- Calls `getMarketMirrorVerticalProfile_` from 14
- Called by 16

### `14_market_mirror_profiles.js`
Market Mirror vertical profile definitions.
- `getMarketMirrorVerticalProfile_` — returns profile with: `metric_labels`, `rep_prompt_groups`, `banned_phrases`, `traditional_effort`
- Supports verticals: auto_retail, hvac, roofing, local_service
- Called by 13 and 16

### `15_market_mirror_narratives.js`
Market Mirror narrative generation.
- `generateMarketMirrorNarrative_` — routes to vertical-specific generators
- Returns `{ headline, subhead, sections: { market, mirror, cost, shift, path, traditional } }`
- Called by 16

### `16_market_mirror_engine.js`
Market Mirror payload orchestrator.
- `buildMarketMirrorPayload_(input)` — full pipeline: normalize (12) → profile (14) → score (13) → narrative (15) → `buildRepSupportSheetData_`
- Called by 11 (sidebar) and 19 (web app)

### `17_market_mirror_renderers.js`
Market Mirror HTML output.
- `renderMarketMirrorHtml_` — produces self-contained branded HTML report
- `renderRepSupportSheetHtml_` — produces rep support data view
- Uses NeoLocal brand palette + Bebas Neue/Barlow typography
- Called by 11 and 19

### `18_market_mirror_test_harness.js`
Editor-only test runners for Market Mirror.
- `getMarketMirrorSampleInput_` — sample inputs for auto_retail, hvac, roofing
- Called by 11 (`getMarketMirrorHtmlForSidebar`) as fallback sample data

### `19_rep_webapp.js`
Web app entry point and API layer.
- `doGet(e)` — routes to 5 modes: rep dashboard, lead detail, add lead, Market Mirror report, rep support sheet
- Server functions called from HTML via `google.script.run`: `lookupManualLeadFromSerpApi`, `saveManualLead`, `saveRepLeadUpdate`
- Also: `fetchSerpApiPlaceByPlaceId_`, `normalizeSerpApiPlaceDetails_`, `buildMarketMirrorInputFromLeadRow_`, `parseMapsIdentifier_`
- Local header utilities (parallel to 06, normalized keys): `buildHeaderIndex_`, `normalizeHeaderKey_`, `getCellByHeader_`, `setCellByHeader_`, `hasHeader_`

### `20_rep_dashboard.html`
Rep dashboard template.
- Template vars: `rep`, `leads[]`, `appUrl`
- CSS grid of lead cards with search/status/task filters
- Client-side JS: task overdue/due-today decoration, freshness (fresh ≤2d / warm ≤7d / stale >7d) badges

### `21_rep_lead.html`
Lead detail and edit form template.
- Template vars: `lead`, `rep`, `appUrl`, `marketMirrorUrl`, `repSupportUrl`
- Full edit form: status, contacts, operator data, tasks, outreach message (read-only + copy), notes history + new note
- Saves via `saveRepLeadUpdate`

### `22_rep_add_lead.html`
Manual lead creation template.
- Template vars: `rep`, `appUrl`
- Paste Maps URL or Place ID → SerpAPI fetch → fill form → save via `saveManualLead` → redirect to lead page

### `crm_action_sidebar.html`
Spreadsheet sidebar template.
- Shows lead context + pipeline action buttons + snooze + "Load Market Mirror" (inline HTML)
- All server calls via `google.script.run`

### `appsscript.json`
- V8 runtime enabled
- Timezone: America/Toronto
- Exception logging: STACKDRIVER

## 4. Data Flow

### Bulk Import Flow
```
Search Config sheet
  → runActiveSearchImports (02)
  → fetchSerpApiLocalResults_ (SerpAPI: engine=google_maps, type=search)
  → normalizeLocalResults_
  → per result:
      buildNormalizedLeadObject_
      buildMarketContext_
      calculateCompetitorSignals_
      generateSnapshot_ (03)
        → buildSnapshotMetrics_ (momentum, undervalued)
        → calculateLeadScores_ (6 scores)
        → determineDiagnosisState_
        → buildSnapshotNarrativePackage_ (04)
        → [vertical routing via 07]
  → appendNewLead_ or updateExistingLeadFromImport_
  → refreshCRMExecutionLayer (10)
```

### Manual Lead Flow (web app)
```
22_rep_add_lead.html
  → lookupManualLeadFromSerpApi (19)
  → fetchSerpApiPlaceByPlaceId_ (SerpAPI: type=place)
  → normalizeSerpApiPlaceDetails_
  → [user fills form]
  → saveManualLead (19)
  → generateSnapshot_ (03) + outreach/reply generation
  → appendNewLead_ (06)
  → redirect to lead detail
```

### Rep Webapp Flow
```
doGet(e) (19)
  mode=rep_dashboard  → getAssignedLeadsForRep_ → 20_rep_dashboard.html
  mode=lead_detail    → getLeadRecordByLeadId_ → 21_rep_lead.html
  mode=add_lead       → 22_rep_add_lead.html
  mode=market_mirror  → buildMarketMirrorPayload_ (16) → renderMarketMirrorHtml_ (17)
  mode=rep_support    → buildMarketMirrorPayload_ (16) → renderRepSupportSheetHtml_ (17)
```

### CRM Action Flow
```
Rep clicks action button (sidebar or lead detail)
  → runCRMActionFromSidebar (11) or saveRepLeadUpdate (19)
  → applyCRMActionByLeadId_ (10)
  → find lead row → validate guardrails → writeRowUpdates_ (06)
  → log activity → refreshCRMExecutionLayer (10)
  → rebuild workspace/follow-up/pipeline views
```

### Market Mirror Flow
```
Sidebar "Load Market Mirror" or web app market_mirror mode
  → buildMarketMirrorInputFromLeadRow_ (19) or manual input
  → buildMarketMirrorPayload_ (16)
      → normalizeMarketMirrorInput_ (12)
      → getMarketMirrorVerticalProfile_ (14)
      → calculateMarketMirrorDerived_ (13)
      → generateMarketMirrorNarrative_ (15)
  → renderMarketMirrorHtml_ (17)
```

## 8. Change Rules (Governance)

- **Do not rename sheet column headers** without a migration plan that updates all read, write, and UI layers touching that column.
- **Do not create duplicate global function names.** Apps Script is global scope — duplicates silently shadow each other.
- **Schema changes** (adding/removing columns) must update: setup function in 01, writeback logic, and any rep-facing UI (19, 20, 21).
- **New intelligence fields** must define all four layers before implementation:
  1. Compute layer — which file calculates it and when
  2. Persistence layer — column name in Leads Master, written by which function
  3. Narrative/use layer — how it feeds snapshot, outreach, shadow job, or reply text
  4. UI layer — where it surfaces in dashboard, lead detail, or Market Mirror
- **Web app URL changes** must update `APP.MARKET_MIRROR_WEBAPP_URL` in `00_config.js` and backfill any stored link columns in existing lead rows.

## 9. Naming Conventions

| Convention | Applies to |
|---|---|
| `functionName_()` trailing underscore | Private/internal — not menu-callable |
| `functionName()` no underscore | Public — menu-callable or `google.script.run` target |
| `UPPER_SNAKE_CASE` | Constants (`APP`, `MM.DEFAULTS`, `MM.SCORING`) |
| `snake_case` | Sheet column headers, lead object keys |
| `camelCase` | JavaScript function names, local variables |
| `MM.` namespace prefix | Market Mirror subsystem globals (namespace guard pattern) |
| `nl-kebab-case` | CSS classes (decoupled frontend target) |
